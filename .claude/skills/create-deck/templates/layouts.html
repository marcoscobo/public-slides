CHAPTER 02

Arquitectura

Cómo se ensamblan las piezas y por qué importa el orden.

CONTEXTO

El problema que resolvemos

Los pipelines tradicionales acumulan latencia en cada salto. Con un diseño event-first, cortamos el tiempo de respuesta en un orden de magnitud.

Esto se traduce en decisiones tomadas con datos frescos, no con datos del día anterior.

PRINCIPIOS

Cuatro reglas que no negociamos

  • 01

    Latencia primero

    Optimizamos para el percentil 99, no para la media.

  • 02

    Sin estado oculto

    Todo lo que importa se materializa en eventos auditables.

  • 03

    Reversibilidad

    Cada deploy puede deshacerse en menos de cinco minutos.

  • 04

    Coste explícito

    Cada componente declara su presupuesto de recursos.

IMPLEMENTACIÓN

Definir un consumidor en 7 líneas

async function consume(stream) {
  for await (const event of stream) {
    if (event.type !== "order.created") continue;
    await projection.apply(event);
    await stream.ack(event.id);
  }
}

Idempotente. Si re-procesamos un evento, la proyección absorbe el cambio sin duplicar estado.

Backpressure. for await bloquea hasta que el handler termina.
