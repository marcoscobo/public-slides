EL PROBLEMA

"En mi máquina funciona"

Sin un entorno reproducible, cada nuevo colaborador —o tú mismo en otro ordenador— parte de cero.

Tres ordenadores distintos. Tres versiones de Node. Tres versiones de Python. Tres resultados diferentes para el mismo código.

El tiempo que se pierde en "funciona en local pero no en producción" no es un accidente: es la consecuencia directa de no haber declarado el entorno como código.

“

El entorno es parte del producto. Si no está versionado, no existe.

Principio de desarrollo moderno Infrastructure as Code · DevContainers
LA SOLUCIÓN

El entorno como código

Herramientas, versiones, variables de entorno: todo declarado en archivos que la IA puede leer, entender y modificar.

Un archivo devcontainer.json define exactamente qué software, qué versiones y qué configuración necesita tu proyecto. No hay ambigüedad.

La IA puede leer ese archivo, entender el contexto del proyecto y actuar sobre él. Sin ese archivo, la IA trabaja a ciegas.

CAPÍTULO 02

Docker

La caja que empaqueta tu entorno y lo hace viajar sin romperse.

DOCKER EXPLICADO

Por qué Docker cambia todo

Docker es la maceta con su tierra específica, nutrientes y microclima. Puedes moverla a cualquier lugar y la planta sigue viviendo igual.

Sin Docker: tu app en tu máquina funciona. En el servidor, falla. Dependencias distintas, versiones distintas, sistema operativo distinto.

Con Docker: tu app vive en un contenedor autodescriptivo. El servidor ejecuta ese mismo contenedor. Funciona igual siempre, en cualquier lugar.

CONCEPTOS CLAVE

El vocabulario de Docker

  • 01

    Imagen Docker

    La receta. Define qué software incluye el contenedor: sistema operativo base, lenguajes, librerías, configuración. Inmutable y versionada.

  • 02

    Contenedor

    La receta ejecutándose. Una instancia viva de la imagen. Puedes tener diez contenedores del mismo tipo corriendo en paralelo.

  • 03

    Dockerfile

    El archivo de texto que describe cómo construir la imagen, paso a paso. La IA genera y modifica Dockerfiles por ti si se los pides.

  • 04

    docker-compose.yml

    El director de orquesta. Coordina múltiples contenedores: tu app, la base de datos, el servidor de colas. Todo arranca con un solo comando.

CAPÍTULO 04

Los Agentes de IA

Claude CLI o Antigravity. El motor que convierte instrucciones en código.

CLAUDE CLI

La IA de Anthropic que actúa en tu proyecto

Claude Code (CLI) convierte tu terminal en un agente de desarrollo que puede leer, escribir, ejecutar y razonar sobre todo tu proyecto.

No es un chatbot. Es un agente que actúa: crea archivos, modifica código, ejecuta comandos y razona sobre el contexto completo del repositorio.

Requiere cuenta en Anthropic Console con créditos de API (de pago). Es la herramienta principal del curso para quien tenga licencia.

Contexto de ventana: 200k tokens. Multiarchivo. Soporte para CLAUDE.md y agentes personalizados.

ANTIGRAVITY · GOOGLE

El agente de Google con capa gratuita

Antigravity es el agente de desarrollo de Google. Equivalente funcional a Claude Code, también opera desde la terminal y actúa sobre el repositorio completo.

La diferencia clave: tiene capa gratuita. Para quien no disponga de licencia de Claude, Antigravity es la alternativa recomendada en este curso.

Mismos conceptos, misma mecánica. Los ejemplos del curso se mostrarán primero con Claude y después con Antigravity para que puedas seguir con cualquiera de los dos.

Requiere cuenta de Google. Integración con Google Cloud y Workspace. Compatible con el mismo flujo de trabajo.

VERIFICACIÓN DEL ENTORNO

Cómo saber que todo funciona

  • 01

    code --version

    Devuelve el número de versión de VS Code. Si no reconoce el comando, reinstalar y marcar la opción "Añadir al PATH".

  • 02

    docker --version && docker run hello-world

    El primer comando verifica la instalación. El segundo ejecuta un contenedor de prueba. Si ves "Hello from Docker", todo funciona.

  • 03

    claude --version / antigravity --version

    Confirma que el agente está instalado. Si falla, verificar que Node.js está en el PATH con node --version.

  • 04

    claude "dime hola" / antigravity "dime hola"

    La prueba definitiva: el agente responde en lenguaje natural. Si pide autenticación, seguir el flujo de login de la herramienta correspondiente.

INSTALACIÓN · CLAUDE CLI

Instalar Claude Code

# Paso 1: Verificar Node.js (versión 18+)
node --version

# Si no tienes Node.js, instálalo desde nodejs.org (versión LTS)

# Paso 2: Instalar Claude Code globalmente
npm install -g @anthropic-ai/claude-code

# Paso 3: Autenticar con tu cuenta de Anthropic
claude auth login
# Se abrirá el navegador para completar la autenticación

# Paso 4: Verificar instalación
claude --version
claude "dime hola"

Requisito: Node.js 18 o superior. Si tienes versiones antiguas, usa nvm para gestionar múltiples versiones.

API Key: Necesitas una cuenta en console.anthropic.com con créditos. El login abre el navegador automáticamente.

Windows: Ejecutar en PowerShell o en WSL2 (recomendado para mejor compatibilidad).

INSTALACIÓN · ANTIGRAVITY

Instalar Antigravity (Google)

# Paso 1: Verificar Node.js (versión 18+)
node --version

# Paso 2: Instalar Antigravity globalmente
npm install -g @google/antigravity

# Paso 3: Autenticar con tu cuenta de Google
antigravity auth login
# Se abrirá el navegador para autenticar con Google

# Paso 4: Verificar instalación
antigravity --version
antigravity "dime hola"

Gratuito: Antigravity tiene capa gratuita con límite mensual de tokens. Suficiente para el curso.

Cuenta Google: Necesitas una cuenta de Google activa. El login usa OAuth estándar.

Mismo flujo: Una vez instalado, los comandos son equivalentes a Claude Code. Misma mecánica, diferente proveedor.

BUENAS PRÁCTICAS

Variables de entorno y secretos

Las claves API nunca van escritas directamente en el código. Nunca.

.env — guarda los secretos localmente. Este archivo NUNCA se sube a Git. Añádelo al .gitignore desde el primer día.

.env.example — documenta qué variables necesita el proyecto, con valores de ejemplo pero sin los valores reales. Este sí se versiona.

La IA gestiona este patrón automáticamente si se le indica en el CLAUDE.md: "nunca escribas valores de API key directamente en el código, usa variables de entorno".

PROBLEMAS FRECUENTES

Errores frecuentes y sus soluciones

EL SÍNTOMA

Qué ves

  • Docker no arranca al ejecutar el primer comando
  • Error: address already in use (puerto ocupado)
  • claude: command not found
  • Error: API key not configured
LA SOLUCIÓN

Qué hacer

  • Verificar que Docker Desktop está ejecutándose (icono en la barra del sistema)
  • Cambiar el puerto en docker-compose.yml de 3000 a 3001
  • Instalar Node.js desde nodejs.org primero, luego repetir npm install -g
  • Ejecutar claude auth login de nuevo y completar el flujo en el navegador
TROUBLESHOOTING

Cuatro errores y sus soluciones

  • 01

    Docker no arranca — Docker Desktop debe estar en ejecución

    Docker CLI necesita que Docker Desktop esté corriendo en segundo plano. Busca el icono de la ballena en la barra del sistema. Si no está, ábrelo manualmente.

  • 02

    Puerto en uso — Cambiar puerto en docker-compose.yml

    Si el puerto 3000 está ocupado, edita ports: - 3000:3000 y cámbiala a 3001:3000. El primer número es el puerto de tu máquina.

  • 03

    Claude/Antigravity no autenticado — repetir auth login

    La sesión puede expirar. Ejecuta claude auth login o antigravity auth login y sigue el flujo en el navegador.

  • 04

    npm not found — Instalar Node.js primero

    Visita nodejs.org, descarga la versión LTS e instala. Cierra y vuelve a abrir la terminal antes de reintentar cualquier comando npm.
