EL PROBLEMA

"En mi máquina funciona"

Sin un entorno reproducible, cada nuevo colaborador —o tú mismo en otro ordenador— parte de cero.

Tres ordenadores distintos. Tres versiones de Node. Tres versiones de Python. Tres resultados diferentes para el mismo código.

El tiempo que se pierde en "funciona en local pero no en producción" no es un accidente: es la consecuencia directa de no haber declarado el entorno como código.

“

El entorno es parte del producto. Si no está versionado, no existe.

Principio de desarrollo moderno Infrastructure as Code · DevContainers
LA SOLUCIÓN

El entorno como código

Herramientas, versiones, variables de entorno: todo declarado en archivos que la IA puede leer, entender y modificar.

Un archivo devcontainer.json define exactamente qué software, qué versiones y qué configuración necesita tu proyecto. No hay ambigüedad, no hay "instala esto que se me olvidó decirte".

La IA puede leer ese archivo, entender el contexto del proyecto y actuar sobre él. Sin ese archivo, la IA trabaja a ciegas.

CAPÍTULO 02

Docker

La caja que empaqueta tu entorno y lo hace viajar sin romperse.

DOCKER EXPLICADO

Por qué Docker cambia todo

Docker es la maceta con su tierra específica, nutrientes y microclima. Puedes moverla a cualquier lugar y la planta sigue viviendo igual.

Sin Docker: tu app en tu máquina funciona. En el servidor, falla. Dependencias distintas, versiones distintas, sistema operativo distinto.

Con Docker: tu app vive en un contenedor autodescriptivo. El servidor ejecuta ese mismo contenedor. Funciona igual siempre, en cualquier lugar.

CONCEPTOS CLAVE

El vocabulario de Docker

  • 01

    Imagen Docker

    La receta. Define qué software incluye el contenedor: sistema operativo base, lenguajes, librerías, configuración. Inmutable y versionada.

  • 02

    Contenedor

    La receta ejecutándose. Una instancia viva de la imagen. Puedes tener diez contenedores del mismo tipo corriendo en paralelo.

  • 03

    Dockerfile

    El archivo de texto que describe cómo construir la imagen, paso a paso. La IA genera y modifica Dockerfiles por ti si se los pides.

  • 04

    docker-compose.yml

    El director de orquesta. Coordina múltiples contenedores: tu app, la base de datos, el servidor de colas. Todo arranca con un solo comando.

DEMO EN VIVO

Stack completo: de cero a producción

El profesor instalará VS Code, Docker Desktop y Claude CLI en tiempo real y levantará una primera web en menos de 10 minutos.

PASO 1 — Descargar VS Code desde code.visualstudio.com e instalar.

PASO 2 — Instalar Docker Desktop desde docker.com/products/docker-desktop.

PASO 3 — En terminal: npm install -g @anthropic-ai/claude-code

PASO 4 — Autenticar: claude auth login (se abre el navegador).

PASO 5 — Primer prompt: claude "crea una web hola mundo en Node.js con Dockerfile"

PASO 6 — Construir y ejecutar: docker build -t demo . && docker run -p 3000:3000 demo

PASO 7 — Abrir http://localhost:3000 en el navegador. Resultado: Hola Mundo.

VERIFICACIÓN DEL ENTORNO

Cómo saber que todo funciona

  • 01

    code --version

    Devuelve el número de versión de VS Code. Si no reconoce el comando, reinstalar VS Code y marcar la opción "Añadir al PATH".

  • 02

    docker --version && docker run hello-world

    El primer comando verifica la instalación. El segundo descarga una imagen de prueba y ejecuta un contenedor real. Si ves "Hello from Docker", todo funciona.

  • 03

    claude --version

    Confirma que Claude CLI está instalado globalmente via npm. Si falla, verificar que Node.js está en el PATH con node --version.

  • 04

    claude "dime hola"

    La prueba definitiva: Claude CLI responde en lenguaje natural. Si pide autenticación, ejecutar claude auth login de nuevo y verificar la API key.

  • 05

    npm --version && node --version

    Prerequisito silencioso: Node.js y npm deben estar instalados antes que Claude CLI. Si fallan, instalar desde nodejs.org (versión LTS recomendada).

BUENAS PRÁCTICAS

Variables de entorno y secretos

Las claves API nunca van escritas directamente en el código. Nunca.

.env — guarda los secretos localmente. Este archivo NUNCA se sube a Git. Añádelo al .gitignore desde el primer día.

.env.example — documenta qué variables necesita el proyecto, con valores de ejemplo pero sin los valores reales. Este sí se versiona.

La IA gestiona este patrón automáticamente si se le indica en el CLAUDE.md: "nunca escribas valores de API key directamente en el código, usa variables de entorno".

PROBLEMAS FRECUENTES

Errores frecuentes y sus soluciones

EL SÍNTOMA

Qué ves

  • Docker no arranca al ejecutar el primer comando
  • Error: address already in use (puerto ocupado)
  • claude: command not found
  • Error: API key not configured
LA SOLUCIÓN

Qué hacer

  • Verificar que Docker Desktop está ejecutándose (icono en la barra del sistema)
  • Cambiar el puerto en docker-compose.yml de 3000 a 3001 (o cualquier otro libre)
  • Instalar Node.js desde nodejs.org primero, luego repetir npm install -g
  • Ejecutar claude auth login de nuevo y completar el flujo en el navegador
TROUBLESHOOTING

Cuatro errores y sus soluciones

  • 01

    Docker no arranca — Docker Desktop debe estar en ejecución

    Docker CLI necesita que Docker Desktop esté corriendo en segundo plano. Busca el icono de la ballena en la barra del sistema. Si no está, ábrelo manualmente antes de ejecutar cualquier comando docker.

  • 02

    Puerto en uso — Cambiar puerto en docker-compose.yml

    Si el puerto 3000 está ocupado por otra aplicación, edita la línea ports: - 3000:3000 y cámbiala a 3001:3000. El primer número es el puerto de tu máquina; el segundo es interno al contenedor.

  • 03

    Claude CLI no autenticado — claude auth login

    La sesión de autenticación puede expirar. Ejecuta claude auth login, sigue el flujo en el navegador que se abre automáticamente y verifica que la API key del perfil de Anthropic Console es válida.

  • 04

    npm not found — Instalar Node.js primero

    Claude CLI requiere Node.js para instalarse. Si npm no existe en tu sistema, visita nodejs.org, descarga la versión LTS e instala. Luego cierra y vuelve a abrir la terminal antes de reintentar.
