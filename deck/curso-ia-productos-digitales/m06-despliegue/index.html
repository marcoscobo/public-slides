CAPÍTULO 01

Dónde desplegar

Cuatro plataformas, un criterio claro: cuál usar según el tipo de proyecto.

PLATAFORMAS DE DESPLIEGUE

Las cuatro opciones principales

  • 01

    Vercel — La más simple para Next.js y frontends

    Detecta el stack automáticamente. Plan gratuito generoso. Ideal para este curso. Sin configuración manual.

  • 02

    Railway — Buena para apps full-stack con BD

    PostgreSQL incluido. Deploy desde GitHub. Plan gratuito 500h/mes.

  • 03

    Render — Alternativa a Railway

    PostgreSQL incluido. Plan gratuito disponible. Nota: se duerme tras 15 min de inactividad en el tier gratuito.

  • 04

    Dokploy / Coolify — Autohosteado

    Para quienes quieren control total sobre la infraestructura. Open source, se instala en tu propio VPS. Requiere más configuración pero sin costes de plataforma.

SECRETOS EN PRODUCCIÓN

Variables de entorno: nunca en el código

Lo que en local guardas en .env, en producción se guarda como secretos en el dashboard de la plataforma.

NUNCA: hardcodear claves API directamente en el código fuente.

NUNCA: subir el archivo .env a GitHub — añádelo siempre al .gitignore.

SIEMPRE: crear un .env.example que documente qué variables se necesitan, sin los valores reales.

La IA puede listar todas las variables que necesita tu proyecto: «¿Qué variables de entorno necesita este proyecto para funcionar en producción?»

CAPÍTULO 02

Desplegar paso a paso

De cero a URL pública en menos de 5 minutos con Vercel.

DESPLIEGUE EN VERCEL

Tu primera URL pública en 6 pasos

  1. 01

    Crear cuenta

    vercel.com con GitHub. Sin tarjeta de crédito. Plan gratuito ilimitado para proyectos personales.

  2. 02

    Import Project

    Add New → Import Git Repository. Conecta tu cuenta de GitHub y selecciona el repositorio.

  3. 03

    Configurar framework

    Vercel detecta Next.js automáticamente. Confirmar la configuración sugerida.

  4. 04

    Variables de entorno

    Settings → Environment Variables. Añade las variables del .env.example antes de hacer deploy.

  5. 05

    Deploy

    Clic en Deploy. Build automático: instala dependencias, compila. Logs en tiempo real.

  6. 06

    URL activa

    En 1-2 minutos tienes tuapp.vercel.app con HTTPS activo. Compártela con quien quieras.

El momento más satisfactorio del curso: ver el producto funcionando en una URL pública. Le pedimos a Claude que prepare el repositorio para Vercel antes de hacer el primer deploy.

CAPÍTULO 03

CI/CD y gestión de cambios

Automatizar el despliegue para que cada mejora llegue a usuarios sin fricción manual.

INTEGRACIÓN Y DESPLIEGUE CONTINUOS

Push y listo: cómo funciona el CI/CD

CI/CD elimina el deploy manual. Cada push al branch main actualiza tu producción. La IA puede configurar este pipeline por ti.

Tiempo total: ~2-3 minutos desde el push hasta que los usuarios ven la nueva versión.

ESTRATEGIA DE RAMAS

Nunca romper producción por accidente

Con esta estrategia de ramas, lo que está en main siempre funciona. La IA puede gestionar todo esto si lo defines en el CLAUDE.md.

main → siempre estable, lo que está en producción y funciona al 100%.

develop → integración de features, probamos aquí antes de pasar a main.

feature/nombre → cada nueva funcionalidad en su propia rama, aislada.

“

El código que no llega a tus usuarios no existe. La única métrica que importa es cuántas veces al día puedes hacer deploy con confianza.

Principio del CI/CD Automatiza el deploy desde el primer día. Cada push a main es una decisión consciente.
CAPÍTULO 04

Monitorización y seguridad

Saber cuándo algo falla antes de que los usuarios te lo digan.

MONITORIZACIÓN BÁSICA

Tres herramientas para saber que todo funciona

  • 01

    UptimeRobot — Vigila tu URL cada 5 minutos

    Email automático si tu app cae. Gratis para 50 monitores. Configuración: 2 minutos. Ideal como primera línea de alerta.

  • 02

    Vercel Logs — Tu primera parada ante cualquier error

    Dashboard con logs en tiempo real y errores de build. Accesible desde el panel de Vercel sin instalar nada.

  • 03

    Sentry — Captura errores con contexto completo

    Plan gratuito disponible. Captura errores de JavaScript con stack trace, usuario afectado y contexto. La IA configura Sentry en tu proyecto en minutos.

SEGURIDAD BÁSICA

Cinco cosas que debes tener desde el primer día

  • 01

    HTTPS automático

    Vercel lo configura solo. Tu URL siempre https://. Sin acción requerida de tu parte.

  • 02

    Variables de entorno para todos los secretos

    Nunca en el código, nunca en GitHub. Dashboard de Vercel → Environment Variables.

  • 03

    Rate limiting en tu API

    Protege tu API de abuso y de costes inesperados en OpenAI. La IA puede añadirlo con express-rate-limit en menos de 5 minutos.

  • 04

    Validación de inputs del usuario

    La IA debe validar siempre los datos que llegan del usuario antes de procesarlos o enviarlos a otra API.

  • 05

    Revisión periódica de dependencias

    Pide periódicamente a la IA: «Revisa si hay dependencias con vulnerabilidades conocidas en este proyecto.»

EL CICLO VIRTUOSO

Con IA en cada paso, una persona puede hacer el trabajo de un equipo

DEMO EN VIVO

Un cambio, un push, producción actualizada

Hacemos un cambio pequeño en el código y observamos en tiempo real cómo Vercel lo despliega automáticamente.

1. Abrimos el código del proyecto en el editor.

2. Cambiamos un texto visible en la interfaz (por ejemplo, el título de la página).

3. git add . → git commit -m 'fix: update title' → git push.

4. Vamos al dashboard de Vercel y observamos el build en tiempo real.

5. En 1-2 minutos: refrescamos la URL pública — el cambio está en vivo.
